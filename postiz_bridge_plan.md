# Postiz Publishing Bridge — Integration Plan

## Integration Approach

Keep the current scheduler as the content management layer. Add Postiz Cloud API as a publishing bridge that gets called when a post transitions from `scheduled` to `publishing`.

```
Current flow:
  draft → scheduled → (nothing happens)

New flow:
  draft → scheduled → publishing → [Postiz API] → published / failed
```

The scheduler stores, edits, and manages posts. Postiz handles the actual delivery to Instagram, Twitter/X, Facebook, and LinkedIn.

## Status Flow

| Status | Meaning |
|--------|---------|
| `draft` | Post created, not yet ready |
| `scheduled` | Post approved, waiting for scheduled time |
| `publishing` | Postiz API call in progress |
| `published` | Successfully delivered to platform |
| `failed` | Postiz API returned an error |

## Integration Points (Minimal Changes)

### 1. Publish endpoint in data_api.py

Add a new route that triggers publishing for a specific post:

```python
@data_bp.route('/api/data/social-posts/<post_id>/publish', methods=['POST'])
@require_auth
def publish_social_post(post_id):
    """Publish a post via Postiz API."""
    from postiz_publisher import publish_post, is_configured
    if not is_configured():
        return jsonify({'status': 'error', 'message': 'Postiz not configured'}), 503

    # Fetch the post
    row = query_one(
        "SELECT * FROM social_posts WHERE id = %s AND project_id = %s",
        (post_id, DEFAULT_PROJECT_ID)
    )
    if not row:
        return jsonify({'status': 'error', 'message': 'Not found'}), 404

    # Update status to publishing
    execute(
        "UPDATE social_posts SET status = 'publishing' WHERE id = %s",
        (post_id,)
    )

    # Call Postiz
    platforms = row.get('platforms', [])
    platform = platforms[0] if platforms else 'x'
    result = publish_post(
        content=row['content'],
        platform=platform,
        scheduled_at=row.get('scheduled_at'),
    )

    if result['success']:
        execute(
            "UPDATE social_posts SET status = 'published', published_at = NOW() WHERE id = %s",
            (post_id,)
        )
        return jsonify({'status': 'success', 'postiz': result['data']})
    else:
        execute(
            "UPDATE social_posts SET status = 'failed' WHERE id = %s",
            (post_id,)
        )
        return jsonify({'status': 'error', 'message': result['error']}), 502
```

## How to Get Integration IDs

1. Sign up at [platform.postiz.com](https://platform.postiz.com)
2. Connect Instagram account (Settings > Channels > Add Instagram)
3. Connect Twitter/X account (Settings > Channels > Add X)
4. Go to Settings > Developers > Public API > copy API key
5. Run: `python postiz_publisher.py` — it will list connected integrations with their IDs
6. Set the IDs as environment variables
