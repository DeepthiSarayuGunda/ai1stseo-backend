## Concept

Build an AI-optimized business directory on `ai1stseo.com` — structured so AI models (ChatGPT, Gemini, Perplexity, Claude) cite it when users ask "best dentist in Ottawa" etc. Not a traditional Yellow Pages — think G2/Capterra for local businesses.

## Why Now

- AI search traffic converts at 4.4x the rate of traditional organic search
- 58% of consumers now rely on AI for product recommendations
- Comparative list articles make up ~1/3 of all AI output mentions
- Sites with 32K+ referring domains are 3.5x more likely to be cited by ChatGPT
- No one has built the AI-optimized local/SMB business directory yet

## What Makes It "AI-First"

- Every listing structured as FAQ blocks (AI loves Q&A format)
- Schema markup on every listing (LocalBusiness + FAQPage)
- Comparison pages ("Business A vs B vs C")
- Category pages ("Top 10 X in Y city")
- `llms.txt` file welcoming AI crawlers
- Fast load times (pages under 0.4s get 6.7 AI citations vs 2.1 for slower)
- Content freshness signals (50% of AI-cited content is less than 13 weeks old)

## Architecture

```
Internet --> AWS (lightweight) --> Cloudflare Tunnel --> Homelab Postgres
                                        ^
                                  Scraping/Ingestion
                                  runs on homelab too
```

### Layer 1 — Homelab (Heavy Lifting)
- PostgreSQL with pgvector extension (AI similarity search)
- Separate schemas: listings, rankings, scraping, analytics
- Scraping pipeline: Scrapy/Playwright for Google Maps, Yelp, Yellow Pages, BBB, Canada411
- Ranking engine: review count, category relevance, data completeness, AI citation readiness
- Claude API batch job (nightly) for auto-generating summaries and FAQ blocks

### Layer 2 — Cloudflare Tunnel (Free)
- Connects homelab to AWS domain securely
- No open ports on home network
- Handles SSL automatically
- Survives dynamic IP changes

### Layer 3 — AWS (Lightweight)
```
Route53 --> CloudFront (CDN) --> Lambda --> API Gateway --> Tunnel --> Homelab Postgres
```
- CloudFront caches pages aggressively (24-48h TTL)
- Lambda only fires on cache misses
- Near zero cost at low traffic

### Layer 4 — DNS Setup (Split DNS)
```
ai1stseo.com        --> AWS (keep as is, SES works)
www.ai1stseo.com    --> Cloudflare Tunnel (directory)
api.ai1stseo.com    --> Cloudflare Tunnel (API)
mail/DKIM records   --> AWS SES (unchanged)
```

### Estimated Cost: ~$10-25/month
(vs $100-300/month running everything on AWS)

## Key Pages to Build (Frontend — Amira)

| Route | Purpose | Example |
|-------|---------|---------|
| `/listing/[business-slug]` | Individual AI-optimized profile | `/listing/sample-dental-clinic` |
| `/category/[city]/[type]` | Category listing page | `/category/ottawa/dentists` |
| `/top-10/[category]/[city]` | Top 10 ranking page | `/top-10/restaurants/nepean` |
| `/compare/[a]/[b]` | Side-by-side comparison | `/compare/sample-dental/capital-dental` |

### Each Listing Page Must Include:
1. BLUF summary (40-60 words, auto-generated by Claude API)
2. Quick Facts (founded, category, location, rating, hours, phone, website, AI visibility score)
3. FAQ section (5-8 Q&A blocks — FAQPage schema)
4. Comparison table (vs competitors)
5. AI Citation Status (which AI models cite this business)
6. JSON-LD schema (LocalBusiness + FAQPage)
7. "Claim This Listing" CTA
8. Breadcrumb navigation

## How Each AI Model Works Differently

| Model | Key Behavior | Optimization Strategy |
|-------|-------------|----------------------|
| ChatGPT | 47.9% of citations from Wikipedia-style sources, 87.4% of all AI referral traffic | Structure listings like Wikipedia — factual, neutral, structured |
| Perplexity | Skews toward Reddit + fresh content (< 90 days) | Nightly scraping keeps listings fresh |
| Gemini | Favors E-E-A-T signals, pulls from Google Business Profile | Strong schema markup + review data |
| Claude | Emphasizes safety, conciseness, analytical depth | Well-structured logical content |

Key stat: 90% of ChatGPT citations come from sources outside the top 20 search results — structure matters more than domain authority.

## Technical Requirements

### Schema Markup (every listing page)
```json
{
  "@context": "https://schema.org",
  "@graph": [
    {
      "@type": "LocalBusiness",
      "name": "Business Name",
      "description": "40-60 word BLUF summary",
      "address": {...},
      "aggregateRating": {...},
      "openingHours": [...]
    },
    {
      "@type": "FAQPage",
      "mainEntity": [
        {"@type": "Question", "name": "...", "acceptedAnswer": {"@type": "Answer", "text": "..."}}
      ]
    }
  ]
}
```

### llms.txt File
```
User-agent: GPTBot
Allow: /

User-agent: Google-Extended
Allow: /

User-agent: ClaudeBot
Allow: /

User-agent: PerplexityBot
Allow: /

Sitemap: https://ai1stseo.com/sitemap-ai.xml
```

### Database Schema (key columns)
```sql
ALTER TABLE listings ADD COLUMN ai_summary text;
ALTER TABLE listings ADD COLUMN faq_json jsonb;
ALTER TABLE listings ADD COLUMN schema_markup jsonb;
ALTER TABLE listings ADD COLUMN comparison_tags text[];
ALTER TABLE listings ADD COLUMN last_scraped timestamp;
ALTER TABLE listings ADD COLUMN freshness_score float;
```
